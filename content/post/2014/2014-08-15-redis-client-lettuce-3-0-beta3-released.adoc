lettuce 3.0.Beta 3 is out now. The new redis commands provide in addition to the basic support also data structures and parsers. This is to support common implementation tasks and to ease adoption.

The Beta 3t is a surprise, because I wanted to release the final after Beta 2. Meanwhile, redis added lots of features and in fact, lettuce was not ready for a final yet. I'll state more about the details in a further, final announcement post. For now enjoy the Beta 3.

<!--more-->

== Fixes

* Fixed Connections.closem, Exception was thrown even if right connection object was supplied (https://github.com/mp911de/lettuce/commit/036d1ea0c8b4c453642df89aeaa4066a75acb13d[`036d1ea`])
* Support for qualified clients in case there are unqualified instances (https://github.com/mp911de/lettuce/commit/5b49c8441ba84b8c4496828ba1df9309397db97f[`5b49c84`])
* Fixed NPE on sync cluster connections (https://github.com/mp911de/lettuce/commit/bbba78343e410f06e4b814c70e248ccd745a3f4a[`bbba783`])
* Changed channel handling to aviod NPE (https://github.com/mp911de/lettuce/commit/707b231593455113a1d29c915969e258b2d33d0f[`707b231`])
* Fixed CDI/Spring support and added tests for CDI/Spring (https://github.com/mp911de/lettuce/commit/b5832f946e940ea926ab4bdff01d5c890b326a28[`b5832f9`])

== Enhancements

* Add support for CLUSTER SLOTS command https://github.com/mp911de/lettuce/issues/7[++#++7]

* Add support for extended CLIENT KILL https://github.com/mp911de/lettuce/issues/12[++#++12]
