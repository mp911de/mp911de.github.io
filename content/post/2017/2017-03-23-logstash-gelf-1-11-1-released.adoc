I released today logstash-gelf 1.11.1. This release fixes a bunch of bugs and brings minor enhancements.

<!--more-->

This release contains following changes:

== Enhancements

* Allow configuration of log message parameter inclusion https://github.com/mp911de/logstash-gelf/issues/87[++#++87]
* Allow selective stack trace extraction with filtering on Java-default representation https://github.com/mp911de/logstash-gelf/issues/99[++#++99] (Thanks to https://github.com/mrmanc[@mrmanc])
* Introduce PoolHolder for ThreadLocal-pooled objects https://github.com/mp911de/logstash-gelf/issues/104[++#++104] (Thanks to https://github.com/artemy-osipov[@artemy-osipov])
* Consider 2xx HTTP status codes as success https://github.com/mp911de/logstash-gelf/issues/114[++#++114] (Thanks to https://github.com/madmuffin1[@madmuffin1])

== Bugfixes

* Missing type info in the stacktrace https://github.com/mp911de/logstash-gelf/issues/97[++#++97] (Thanks to https://github.com/mrmanc[@mrmanc])
* Close channel in GelfTCPSender.connect() https://github.com/mp911de/logstash-gelf/issues/107[++#++107] (Thanks to https://github.com/chainkite[@chainkite])
* Guard GelfTCPSSLSender.closeSocketChannel against uninitialized SSLEngine https://github.com/mp911de/logstash-gelf/issues/111[++#++111] (Thanks to https://github.com/badreddine16[@badreddine16])

== Other

* Upgrade to Mockito 2.3 https://github.com/mp911de/logstash-gelf/issues/100[++#++100]
* Remove fest-assert usage https://github.com/mp911de/logstash-gelf/issues/100[++#++100]
* Migrate tests to AssertJ https://github.com/mp911de/logstash-gelf/issues/100[++#++100]
* Migrate tests to JUnit 5 https://github.com/mp911de/logstash-gelf/issues/101[++#++101]
* Drop Java 7 build profile https://github.com/mp911de/logstash-gelf/issues/102[++#++102]
* Upgrade Mockito to 2.7.17 https://github.com/mp911de/logstash-gelf/issues/113[++#++113]

Thanks to all who contributed.
